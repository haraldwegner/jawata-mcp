# Contributing to JAWATA (jawata-mcp)

Thanks for your interest in JAWATA — IDE-grade, compiler-accurate Java intelligence and
behaviour-preserving refactoring for AI agents. A few ground rules keep the project
sustainable and safe to change.

## Licence & CLA

JAWATA is licensed under **AGPL-3.0** (see [`LICENSE`](LICENSE)). Portions of the original base
are retained under MIT (see [`NOTICE`](NOTICE)).

By contributing you agree to the **JAWATA Contributor License Agreement** ([`CLA.md`](CLA.md)),
recorded automatically via **CLA-assistant** on your first pull request. The CLA lets the
maintainer keep the project sustainably licensed (including any future dual-licensing) while
**you retain full rights to your own work**.

## Provenance — sign your commits

Every commit must carry a Developer Certificate of Origin sign-off:

```bash
git commit -s -m "feat(scope): your change"
```

This certifies you wrote the change, or have the right to submit it.

## Attribution policy

Commits are attributed to their **human author only**. Do **not** add co-author or
generated-by trailers for AI / code-generation tools — e.g. no `Co-Authored-By:` lines and
no "Generated by …" notes in commits, PR bodies, code comments, docs, or release notes.
PRs that carry such trailers will be asked to remove them before merge.

## What we look for

- **Behaviour-preserving changes, verified.** Build green (`mvn clean verify`) and the touched
  tests green. A change that alters observable behaviour is a *feature*, not a refactor — ship
  it as its own, separately-tested commit, never smuggled inside a structural move.
- **Small, reviewable PRs** — one logical change each.
- **New detector / tool kinds:** include a focused test fixture (a deliberately-smelly case
  *and* a clean case) and a tool description that self-routes ("use instead of `grep` for X").

## Building

JAWATA is an Eclipse-JDT / Tycho Maven build (Java 21):

```bash
mvn clean verify          # full reactor + tests
```

## Security

Report vulnerabilities **privately** per [`SECURITY.md`](SECURITY.md). Please don't open
public issues for security bugs.
