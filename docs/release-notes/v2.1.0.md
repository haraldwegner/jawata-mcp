# goja-mcp v2.1.0 ‚Äî Knowledge-store durability, maintenance & multi-language (Sprint 21a)

v2.0 shipped the experience store; v2.1.0 makes it **durable, maintainable, and
multi-language** ‚Äî the full Sprint 21a. The `experience` front door grows from 7 to
**15 kinds** (still one tool; 41 tools total).

## Store continuity ‚Äî your knowledge now survives restarts and redeploys

In v2.0 the store was opened inside the launcher's **session-isolation directory** ‚Äî a random
subdir created per resident start and **deleted on clean shutdown**. Recorded knowledge was
orphaned on every redeploy and destroyed on every clean restart.

- **User-level shared store is the new default:** `${XDG_DATA_HOME:-~/.local/share}/goja/`
  `experience.mv.db`. Knowledge belongs to the user, not the workspace ‚Äî methodology and
  domain entries are recallable from *every* workspace, while symbol/package scope-containment
  keeps repo-specific recall self-scoped exactly as before.
- **AUTO_SERVER:** multiple residents (different workspaces, or concurrent sessions of one)
  attach to the same store transparently.
- **Modes** via `-Dgoja.experience.store`: `shared` (default) ¬∑ `workspace` (per-workspace
  file at the stable workspace root ‚Äî never the session dir) ¬∑ `memory` ¬∑ an explicit
  directory path. The launcher now publishes the original `-data` path
  (`goja.workspace.root`) so the stable root is authoritative.
- **One-time recovery:** on startup the resident sweeps existing orphaned session stores and
  imports their entries (newest first, deduplicated, symptoms/links preserved, idempotent via
  markers; source files are never deleted).

## Versioned schema + migrations ‚Äî the store can now evolve

v2.0's `CREATE TABLE IF NOT EXISTS` could create a store but never change one.

- **`schema_version` + ordered, additive-first migrations**; a real migration writes an H2
  online backup zip beside the store first.
- **Migration v2 adds facets:** `workspace_id` / `project_id` (provenance, stamped from
  `workspace.json` on every write) and `language` (default `java`) ‚Äî the groundwork for the
  shared store and for multi-language safety (Wave 2).
- **From-the-future guard:** a store written by a newer resident is refused read-write with a
  clear message ‚Äî and the server keeps running on a non-persistent in-memory store instead of
  dying.

## The store finally reads your whole memory ‚Äî `load` defaults + the link graph

- **No-path `load`** seeds from default roots: `-Dgoja.memory.roots` extras, the layered
  `CLAUDE.md` set (global `~/.claude` + each project dir and its ancestors up to `$HOME`),
  and the Claude per-project memory dirs.
- **Link-graph crawl:** ingest follows `[[wikilinks]]` and relative `[x](file.md)` links
  transitively ‚Äî how memory is actually organized (an index plus cross-linked fact files).
  Cycle-safe, bounded (depth/files/bytes, tunable via `-Dgoja.memory.max*`), and skips are
  **reported, never silent**. `recursive:true` additionally walks subdirectories.
- **`reseed`** ‚Äî the explicit initial load: atomic wipe ‚Üí load-from-defaults, hard-gated on
  `confirm:true` (it refuses *before* wiping when no roots exist).

## Curation + hygiene verbs

- **`list`** ‚Äî browse by type / status / scope / language; INCLUDES rejected/superseded
  (you cannot promote what you cannot see). `format=text` for hook-safe lines.
- **`export` / `import`** ‚Äî full-fidelity dump (facets, symptoms, links, timestamps) inline
  or to a JSON file; lossless round-trip, dedup by id. Real backup + cross-machine seeding
  ahead of the networked store.
- **`prune`** ‚Äî GC aged rejected/superseded (`days`, default 30).
- **`dedup`** ‚Äî surface near-duplicates; `confirm:true` merges (best survives ‚Äî accepted
  beats candidate, then newest; losers superseded, never deleted).
- **`compact`** ‚Äî reclaim H2 file space (brief close + transparent reopen).
- **`stats`** ‚Äî counts by status/language + store file/size (drives the studio Knowledge view).

## Multi-language safety

`refresh` used to supersede EVERY non-resolving anchor ‚Äî a JDT resolver can never see a
Rust/TS anchor, so one refresh silently killed all non-Java knowledge. Entries now carry a
**`language`** facet (default `java`); only Java anchors are judged by JDT at refresh and
ingest ‚Äî everything else is opaque and safe. `record` accepts `language`; `load` reads
frontmatter `language:`.

## Build resilience

Target-platform URLs are parameterized (`p2.base`): canonical `download.eclipse.org` by
default, one-flag fallback to the official FAU mirror, and CI preflights the primary and
flips automatically during outages (p2 has no client-side metadata failover of its own).
