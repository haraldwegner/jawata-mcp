# jawata-mcp v3.4.0 — the store retrieves by meaning

The experience store has always been able to answer a question phrased the way
the knowledge was written down. This release makes it answer a question phrased
the way you actually asked it.

jawata now carries its own embedding model — bundled, run by our own inference,
no network, no external process, no API key. A cue is turned into a vector and
compared against the store; entries whose MEANING is near yours are nominated
alongside the ones keyword matching finds. The two are a union, never a
replacement: an exact symbol name still resolves deterministically, and a
paraphrase now finds what it always should have.

Measured on the real corpus with twelve questions whose right answers were
frozen before any retrieval code existed: **keyword answers 1 of 12, the shipped
union answers 11 of 12**.

## What is different when you use it

**A question in your own words finds the lesson.** "Public methods on a type
have no documentation" now reaches a note filed under entirely different
wording. Nothing about how you record knowledge changes.

**Facts and experience are answered differently, on purpose.** A fact about code
is address-bound: it passes an exact check or it is not returned, and a fact
whose symbol is gone says so. Experience is offered as *analogy* — capped at
two, labeled with why it surfaced ("same operation", "similar symptom",
"meaning-near") and where it was learned, framed as something to judge rather
than obey. **No similarity score appears anywhere in any answer**, because a
number invites being treated as authority, and an analogy is not one.

**Experience whose code is gone still answers.** The anchor is provenance, not a
criterion — a lesson learned on a class that no longer exists is still true, and
is now returned as "learned on X, since removed" instead of being flagged stale.

**The precedent steer gained a second tier.** When a tool went wrong on *this*
target before, it warns and charges a justification exactly as it did in v3.3.1
— unchanged. When something similar went wrong on a *different* target, you get
one advisory line, clearly labeled as a different case, and nothing is charged.

**Recording a paraphrase of an existing entry tells you so.** The entry is
stored either way — a dedup that can lose knowledge is worse than a duplicate —
and the response proposes a merge. The threshold was derived from 84
hand-labeled pairs of the real corpus, not guessed: every pair it flags was a
true duplicate in that set.

**A past seat run answers a question shaped like it.** Ask something resembling
work a seat did before and you get back which seat ran, what the human decided,
and how it ended. A run nobody has judged yet says "not yet judged" rather than
staying silent.

**`health_check` now names the embedder.** Which matrix backend actually loaded,
which model produced your vectors, and — when semantic recall is off — why.

**`experience(kind=stats)` gained a quality block.** Counts of what the recall
system is doing: which surfaces fire, whether steers are followed or paid past,
what happened afterwards, and how each fit-gate criterion behaves. Every number
carries the sentence saying it is correlation and thin at the start.

## If the embedder cannot load

Everything degrades to exactly v3.3.1 behaviour: keyword and alias matching,
identity-only precedent, no analogies. The resident starts, `health_check` says
semantic recall is off and why, and nothing silently pretends to work.

## Sizes and speed

The bundled weights add **45.4 MB** (fp16, chosen by measurement — parity
against the fp32 reference is 0.999999). A single cue embeds in ~29 ms with the
Vector API and ~91 ms without.

The studio launches the resident with `--add-modules jdk.incubator.vector` when
that module exists. Without it the scalar path produces identical vectors,
slower — the flag is a speed decision, never a correctness one, and
`health_check` reports which one is running.

## Compatibility

- The store migrates to schema v7 additively on first open. Existing entries
  keep working immediately and are embedded in the background.
- No new tool. `experience(kind=recall)` accepts one optional new argument
  (`surface`), used only for the quality counters.
- Tool count unchanged at 45.
