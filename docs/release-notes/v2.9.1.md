# jawata v2.9.1 — dogfood patch on v2.9.0 (self-workspace load + store safety)

Same-day dogfood of the 2.9.0 fleet found the resident **blind to jawata's own
source** — and a knowledge-store safety gap that blindness triggered.

## D1 — project loader: nested aggregators + `<sourceDirectory>` interpolation

The post-de-Tycho repo shape (root pom → `build/` aggregator → leaf modules whose
`<sourceDirectory>` points back into the bundle dirs) loaded with **0 source files**:

- Module traversal is now **recursive** (cycle-safe, depth-capped) instead of
  one level deep.
- `${project.basedir}` / `${basedir}` in `<sourceDirectory>` /
  `<testSourceDirectory>` are interpolated before resolution (the literal text
  silently failed the directory check).
- The Maven classpath probe writes a **relative, per-module** output file and
  merges the results. The old absolute file was overwritten by every reactor
  module — the last, dependency-less aggregator left it empty
  (`-Dmdep.appendOutput` is a no-op in maven-dependency-plugin 3.7.1).

Self-workspace after the fix: **471 source files, 33 packages, 68 classpath
entries, 0 compile errors** (was: 0 files; or 7938 errors via the PDE-add
workaround).

## D2 — experience store: mass-stale circuit breaker

The startup auto-refresh judged every symbol anchor against the (D1-broken,
empty) workspace and **superseded every self-anchored entry** — 22c/22d domain
facts, dogfood lessons, all silently dropped from recall. New guard: when ≥ 3
anchors are judged and **not one** resolves, the workspace is treated as broken
evidence (`workspace_suspect` in the refresh report) and **no status is
changed**. Absence of evidence is not evidence of absence. One positive verdict
disarms the breaker; genuine staleness still supersedes.

Recovery for stores hit by 2.9.0: re-promote the wrongly superseded entries
(`experience(kind=promote, id=…)`) once the fixed resident has loaded the
workspace.
