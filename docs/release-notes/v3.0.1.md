# jawata v3.0.1 — the audit patch on v3.0.0

The standing rule: every release is followed by a dogfood-in-anger round and
a fresh-context implementation audit of the released code against the spec's
deliverable bodies. Three adversarial auditors reviewed v3.0.0; the mcp side
came back KEEP with two edge findings, fixed here.

## Fixed

- **A dead async-sweep worker could report "running" forever.** The
  background family-sweep worker caught `RuntimeException` only; an `Error`
  (OOM, stack overflow) skipped the completion handshake and `status`
  showed `running` indefinitely. The worker now catches `Throwable` — a
  failed sweep always finishes into a loud, retrievable error.
- **A completed sweep could be mislabeled `partial`.** A cancel that landed
  while the LAST detector was already running let the sweep complete fully,
  yet the result carried `state: cancelled, partial: true` with
  `kindsDone == kindsTotal` — internally contradictory. `cancelled` is now
  derived from whether the loop actually stopped early, so a complete
  result is always reported complete.

Same 45 tools; no schema changes. Serial suite 1435/1435.
