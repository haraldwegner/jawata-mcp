# goja-mcp v1.6.0 — The guide at the protocol level + workflow-collapse (Sprint 22, PRE layer · tool half)

The *tool half* of Sprint 22's PRE layer. Where **goja-studio v1.4.0** enforces "try goja
first" from the client-side hook, v1.6.0 makes goja itself carry the guide — into every
client, over the connection — and removes the multi-call friction that makes falling back to
`grep` tempting in the first place. **Zero net new tools** — the surface stays at **40**.

## The guide travels with the connection — MCP `instructions`

`initialize` now returns the protocol-level **`instructions`** field: a tight trigger→tool
guide (search_symbols / find_references / analyze / inspect / rename_symbol / move / extract
/ `generate(kind=copy_class)` / `refactoring(action=plan)`), framing text search as the
fallback for non-Java / non-semantic content only. Clients that honour the field (Claude
Code and others) inject it into the agent's context at connect — so "use GOJA first, not
grep or a hand-edit" ships with goja itself, independent of any deploy-written rule block,
in every client.

## Workflow-collapse — `analyze(kind=symbol)`

"Understand this symbol" was three calls (definition → type → references). Every hop back to
the agent is a chance to defect to `grep`. v1.6.0 folds them into one:

- **`analyze(kind=symbol, filePath, line, column)`** returns `{ definition, symbol,
  references }` in a single call, composing `go_to_definition`,
  `get_at_position(kind=symbol)` and `find_references` (references capped for a bounded
  payload).
- Best-effort merge: a section is omitted when its delegate finds nothing; the call fails
  only when the position resolves to nothing at all — then it returns the definition
  lookup's error, which carries the `search_symbols` recovery hint.
- Surfaced only as a kind on the existing `analyze` front door — **not** a new registered
  tool.

## Compatibility

No breaking changes. `instructions` is additive to the handshake; `analyze` gains one kind
(`symbol`), and every existing kind, tool, and action is unchanged. Tool count remains **40**.
