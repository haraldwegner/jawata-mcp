# GOJA MCP v2.2.3 — the push channel reaches your memory corpus

Point release on v2.2.2. Live dogfood on a real 97-entry store showed the data was all
there — and the automatic channels retrieved none of it. Three content-pipeline fixes:

- **The primer injects your standing rules.** The SessionStart primer filtered for
  `domain_fact`-style types only; a real md corpus maps to `feedback`/`user`/
  `naming_convention` — so 97 entries produced an empty primer. Those standing
  how-to-work types are now primer-eligible (references, projects and notes stay
  cue-gated).
- **Symptom recall is tokenized.** A cue like `blank webview` used to match only as one
  contiguous substring and missed summaries where the words are non-adjacent
  ("…webview content area stays blank…"). Each cue token now matches independently
  against symptoms and summaries — in the query and in the fit gate. Terminal-or-absence
  semantics unchanged; all tokens must fit.
- **No more junk rows from index files.** `MEMORY.md`-style indexes and contentless
  files are treated as link hubs: their links are crawled, nothing is ingested.
  `CLAUDE.md`-style files (no frontmatter, body full of rules) derive their summary
  from the first content line instead of the filename.

After updating: run one **Load** (changed summaries re-ingest, junk rows disappear on
the next wipe+load or age out) — then the SessionStart primer carries your actual rules
and symptom recall finds real answers.
