# goja-mcp v1.2.0 — Fowler smell detection (Sprint 17)

The first content release of the autonomous-engineering pipeline's **detect** head:
GOJA can now find *where* and *why* to refactor, not just *how*. Eighteen Fowler smell
detectors land as **`find_quality_issue` kinds** — registered into the Sprint 16b
`DetectorCatalog` and projected into the tool's `kind` enum. **No new tools**: the loaded
MCP surface stays at **39**; the `kind` enum grows **8 → 26**. This is the 16b "register a
kind, not a tool" invariant proven at scale.

## New smell kinds for `find_quality_issue(kind=…, threshold?)`

Size / shape:
- **`long_method`** — LOC over `threshold` (default 40) or cyclomatic complexity > 10 → Extract Method.
- **`god_class`** — large (members > `threshold`, default 20) **and** high fan-in (≥ 8 referencing
  types) → Extract Class. Distinct from the size-only `large_classes`.
- **`long_parameter_list`** — > `threshold` parameters (default 4) → Introduce Parameter Object.
- **`data_clumps`** — the same 3+-parameter tuple recurring across ≥ `threshold` methods (default 2)
  → Extract Class / Parameter Object.

Relational / behavioural (binding-resolution based — a new class of detector for GOJA):
- **`feature_envy`** — a method using one foreign type's members more than its own → Move Method.
- **`message_chains`** — call chains longer than `threshold` (default 3) → Hide Delegate.
- **`inappropriate_intimacy`** — two classes each accessing the other's fields → Move Method/Field.
- **`middle_man`** — a class mostly delegating to a field (≥ `threshold`%, default 50) → Remove Middle Man.

Type code:
- **`primitive_obsession`** — parameter lists dominated by primitives/String → Replace Type Code with Class.
- **`switch_statements`** — switch over an int/String type code (≥ `threshold` cases, default 3;
  enum switches excluded) → Replace Conditional with Polymorphism.
- **`refused_bequest`** — an `@Override` that just throws `UnsupportedOperationException`
  → Replace Inheritance with Delegation.
- **`temporary_field`** — an instance field used by only one method → Extract Class.

Structure / generality:
- **`lazy_class`** — a small standalone class with low fan-in → Inline Class.
- **`speculative_generality`** — an interface/abstract class with exactly one subtype → Collapse Hierarchy.
- **`parallel_inheritance`** — two hierarchies varying together (`<Variant><BaseName>`) → Move Method/Field.
- **`incomplete_delegation`** *(fork-original, REFACTORING_LESSONS_LEARNED §7)* — a loop recovering an
  element's identity via a getter chain on a collaborator (the O(n) scan that means the object lacks
  its own state) → give the object its state + a key→items index (Extract Class).

Change correlation:
- **`divergent_change`** — a file changed in many commits across many areas → Extract Class.
  *(git-history based; no-op off a git work-tree.)*
- **`shotgun_surgery`** — a type referenced from many distinct types (a change ripples widely)
  → Move Method/Field to gather the responsibility.

## Under the hood
- New `org.goja.mcp.tools.smell` package: `MethodMetrics` (extracted from `GetComplexityMetricsTool`,
  behaviour-preserving), `AbstractAstDetector` (file-walk + parse + severity-sorted `Findings`),
  `FowlerDetectors` (catalog contributor), and a graceful `GitHistory` (`git log`) capability.
- Each detector ships a deliberately-smelly **and** a clean fixture with a flag/ignore test;
  results are surfaced as the canonical `Finding` shape (`{kind, filePath, line, severity, message,
  symbol}`).

## Compatibility
Additive. Existing `find_quality_issue` kinds and every other tool are unchanged; `health_check`
tool count stays 39. Most smell kinds accept an optional `threshold` to tune sensitivity.
