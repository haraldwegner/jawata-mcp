# jawata-mcp v3.3.0 — the intelligent injector, redesigned (Sprint 26a)

Sprint 26 shipped seven ML "learners", six of them hollow. Sprint 26a is the
redesign: the injector is **capability awareness + a deterministic experience
loop + defined rules — zero ML models.** The agent reasons from precedent over
an outcome-labeled store; it does not learn weights.

## The experience loop (capture → retrieve → surface)

Every tool outcome that carries a lesson is captured, and a later similar
situation gets the precedent surfaced on the answer:

- **Selective capture** — a mutate's compile result (`compiled` / `reverted`)
  or a tool error lands in a new `tool_experience` store lane. A routine
  successful read/search captures nothing (the tool always answers — no
  experience, no noise). A failed write is loud, never a silent drop.
- **Baseline retrieval + a weighted steer** — when you work on a target that
  has past outcomes, the answer carries `⚠ PRECEDENT — <tool> was reverted/
  errored N× in a case like this; prefer what worked, or note why`. Retrieval
  sits behind a seam so Sprint 27 swaps in embeddings without touching capture
  or surfacing.

## Capability awareness

The MCP `instructions` (delivered at session start to every client) now carry
**deep how-to + pitfalls** for the reflex-relevant tools, not a bare list:
stage a refactor with `auto_apply=false` and review the diff; `debug` reads
live values with **zero code change** (vs hand-added logging); `profile`
samples the JVM and names the hotspot as a symbol with **zero code change**
(vs a hand-rolled stopwatch).

## The architect-involvement gate (deterministic)

A mutate delta that introduces a smell, changes a signature/hierarchy, or
exceeds a line threshold (~500, configurable) is flagged for architect review
— a staged edit is reviewable before it lands. A plain edit passes. This fixed
rule replaces Sprint 26's edit-switch model.

## The edit-ML retired — zero ML models

`OnlineLogreg`, `HandTree`, `Learner`, `RollingRecord`, `FeatureVector`,
`LearnerService` are deleted. `experience(kind=train|learner_status|
observe_edit)` stay valid and answer honestly — `status: retired`, `models:
[]`, `replacedBy: [the experience loop, the architect gate]`, with the live
capture count. Not an empty-result lie, not an error.

## Also

- **Store resilience (F1)** — `LOCK_TIMEOUT` + a widened retry budget so the
  shared store no longer drops writes when two residents use it at once.
- toolCount unchanged at **45**.
