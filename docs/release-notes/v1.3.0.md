# goja-mcp v1.3.0 — SOLID violation detection (Sprint 20)

The second detection content release. SOLID violations ship as **`find_quality_issue`
kinds** under a new **`family`** lens — tool count stays **39**, the `kind` enum grows
**26 → 30**. The pivotal design: **families are tags, and SOLID mostly re-frames shipped
Fowler smells rather than re-detecting them.**

## The `family` filter (quality / fowler / solid)
`find_quality_issue` gains an optional `family`:
- `family="solid"` with no `kind` **runs the whole SOLID set and merges the findings** —
  the SOLID violations as a unit.
- `kind` + `family` validates membership; a single `kind` works as before.

A `kind` can belong to several families, so the overlap between SOLID and Fowler is resolved
by **tagging**, not duplication: `incomplete_delegation` (SRP), `refused_bequest` (LSP), and
`divergent_change`/`shotgun_surgery` (the OCP trace) are tagged `solid` with zero new code.

## New SOLID kinds
- **`dip`** (Dependency Inversion) — a field of a concrete type used only via methods of an
  interface it implements; depend on the abstraction. Conservative (a single interface must
  fully cover the field's usage).
- **`isp`** (Interface Segregation) — a fat interface (≥ `threshold` methods, default 6) whose
  callers use disjoint method subsets; split it. Clusters methods by shared caller.
- **`srp_cohesion`** (Single Responsibility / low cohesion) — a class whose methods split into
  ≥2 disjoint field-usage clusters (LCOM4). Complements the §7 unfinished-encapsulation SRP
  signal (`incomplete_delegation`). Conservative: ignores constructors, accessors, and
  equals/hashCode/toString; requires ≥2 fields and ≥ `threshold` real methods (default 4).
- **`lsp`** (Liskov Substitution) — an `@Override` whose body just throws, rejecting the
  inherited contract; not substitutable. **Broader than `refused_bequest`** (any exception,
  not only `UnsupportedOperationException`).

## SRP and OCP — by design
- **SRP** is anchored on the `REFACTORING_LESSONS_LEARNED.md` §7 "unfinished encapsulation"
  pattern (a responsibility held by a collaborator instead of the object) — already shipped as
  `incomplete_delegation`, now tagged `solid`. `srp_cohesion` adds the complementary LCOM view.
- **OCP** has **no detector**: an OCP violation isn't mechanically detectable without the
  intended extension axis (design intent), and a churn heuristic would reintroduce false
  positives. Its observable trace is the tagged `divergent_change`/`shotgun_surgery`; its cure
  (introduce the abstraction) is a refactoring goal in Sprint 19.

## Under the hood
- `SolidDetectors` contributor (family `solid`), composed into `FindQualityIssueTool`'s ctor.
- `SmellSearch.referencingTypeCount` extracted from the three copy-pasted fan-in helpers
  (god_class/lazy_class/shotgun_surgery), behaviour-preserving.

## Compatibility
Additive. Existing tools and the 26 prior kinds are unchanged; `health_check` tool count stays
39. New `family` filter and four new kinds; most accept an optional `threshold`/`includeTests`.
