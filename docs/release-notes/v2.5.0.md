# GOJA MCP v2.5.0 — Symbol anchors + grounded recall (Sprint 21e)

The store's facts now point at code without hand-written frontmatter, recall carries
the classify-against-closed-set contract, and `get_diagnostics` finally reports
problems for closed files.

## Ingest-time symbol-anchor resolution (loader v5)

A lesson whose text names its code (`SlotManager.freeSlot`, `Slot#activeOrder` — the
way real postmortems are written, unqualified and mixed-notation) now anchors itself:

- **Backticked tokens are parsed** (`Type`, `pkg.Type`, `Type.member`, `Type#member`,
  args stripped; `.`/`#` member notation collapses into one candidate) and
  **JDT-resolved**. A token anchors ONLY when it resolves **uniquely to a
  project-SOURCE type** — dead/renamed names don't resolve, JDK/library types are
  binary, ambiguous simple names (nested types included) are refused. Absence beats a
  wrong pointer, everywhere.
- **Type-level dominance** picks the entry's ONE anchor (strict mention max; tie →
  none); the member is appended only when the mentions agree on exactly one member
  that actually exists on the resolved type.
- **Section entries anchor from their own text** — previously they could carry no
  symbol at all (frontmatter can't reach them). This closes the gap where a lesson
  written for `freeSlot` was invisible to the symbol channel the PreToolUse refactor
  hook queries.
- **Auto-anchors are column-only.** The frozen entry body never gains a `symbol` key —
  that key remains the author's asserted-provenance marker. Frontmatter `symbol:` and
  agent-recorded `record(symbol=…)` anchors keep today's semantics unchanged, wins
  included.

## Anchor lifecycle

- **Backfill**: memory usually loads before projects — after every project load
  (startup auto-load AND tool-initiated `load_project` / `project(add|remove)`),
  NULL-anchor entries get one resolution pass; hits write the column only, so
  byte-strict skip-unchanged never re-ingests.
- **`refresh()` provenance split**: an unresolvable ASSERTED anchor still supersedes
  the entry (the author asserted it); an unresolvable AUTO anchor is **cleared, never
  used to supersede** — the lesson outlives its pointer, and backfill may re-anchor it
  against the next project set.
- **Recall matches at type level regardless of member notation**: a type-level anchor
  answers a member cue and a member anchor answers its type's cue (`#` enclosure in
  both the SQL candidate gather and the fit gate).
- `LOADER_VERSION` 4→5: existing stores self-heal on their next load — one re-ingest
  spike, then byte-strict silence, anchors appear retroactively.

## The classify contract in recall (Recall-Gap §5.5)

Every recall MATCH now steers: *"Match the observation to ONE of these with evidence,
or declare it genuinely new — do not generate a novel cause."* — in `meta.steering`
only. The agent's diagnosis task flips from generating a plausible cause to
classifying against the returned closed set. Absence responses keep the generic
steering; the hook-parsed `format=text` payload is byte-identical.

## `get_diagnostics` honesty

`cu.reconcile()` returns a null AST for non-working-copies — in a headless server no
file is ever a working copy, so the tool had **never** reported problems for closed
files. It now reads the file's post-build PROBLEM markers (per-call fresh since
v2.4.0's strict disk sync), same response shape. A closed file with a syntax error
finally shows it.
