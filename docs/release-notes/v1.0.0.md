# goja-mcp v1.0.0

**GOJA** — compiler-accurate Java intelligence for AI agents. The first public release, and the
start of a fresh version line. goja-mcp is an Eclipse-JDT–backed MCP server that gives AI agents
IDE-grade Java analysis, navigation, and **behaviour-preserving refactoring** on real-world
workspaces — the depth a human gets in Eclipse or IntelliJ, packaged for agents over MCP.

## What's in the box (85 tools)

- **Navigation & search** — `find_references`, `get_call_hierarchy_*`, `find_implementations`,
  type hierarchy, `search_symbols`, parametric `find_pattern_usages(kind)`.
- **Analysis** — control/data flow, complexity, change-impact, naming + Javadoc + null-safety
  analysis, `find_quality_issue(kind)`.
- **Refactoring with one-call undo** — rename, extract (method/var/const/interface), inline,
  move class/package, pull-up/push-down, encapsulate-field, change-signature; every mutating
  tool applies by default and returns an `undoChangeId`.
- **Modernisation, codegen, dependency management** (Maven + Gradle), workspace-wide
  `compile_workspace`, duplicate-code detection + removal, multi-project workspaces with
  live `workspace.json` reconciliation.

## The rebrand (from javalens-mcp v1.12.0)

- Java packages `org.javalens.*` → `org.goja.*`; runtime jar `javalens.jar` → `goja.jar`.
- MCP server identity → **GOJA**; environment variables `JAVALENS_*` → `GOJA_*`.
- Version line **reset to v1.0.0** — a clean break, not a continuation of the javalens v1.x line.

## Install

Use **[goja-studio](https://github.com/haraldwegner/goja-studio)** — the desktop control plane
that downloads the GOJA runtime, manages workspaces, and writes the MCP entry into your agent's
config (Cursor / Claude / Antigravity / IntelliJ):

```bash
curl -sSL https://raw.githubusercontent.com/haraldwegner/goja-studio/main/install.sh | bash
```

## Migrating from javalens-manager

Install goja-studio — on first launch it migrates your config/state/cache directories
(`…/javalens-manager` → `…/goja-studio`) so workspaces and settings carry over, downloads the
goja-mcp v1.0 runtime, and continues to recognise previously-deployed `jl-`/`javalens-` MCP
entries for clean reconciliation.
