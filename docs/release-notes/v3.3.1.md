# jawata-mcp v3.3.1 — the precedent push, as specified

v3.3.0's precedent push shipped **narrower than the signed Sprint-26a D2
contract**. An external code review against the spec found two gaps — the
internal audits had checked the deliverable's one-line measure, which the
weaker implementation did satisfy. Both are closed here.

## The push carries the how-to

The D2 body says the injector "surfaces the precedent **(with the how-to)**".
v3.3.0 named the tool and the agreement count and stopped there:

```
⚠ PRECEDENT — `extract` was reverted/errored 1× in a case like this. …
```

Now the steer ends with that tool's one-line how-to — for `extract`, *stage it:
pass `auto_apply=false`, review the diff, then apply*; for `rename_symbol`,
*updates ALL references — address by FQN*. A precedent you cannot act on is a
name, not a steer. The lines are phrased to match the catalog delivered at
session start, so the two cannot drift apart.

## The justification-cost is levied, not merely worded

The D2 body: a steer "with a **justification-cost to defect** — not an optional
hint the agent may ignore … **enforced-by-default**; doing otherwise costs a
written justification". v3.3.0 said exactly that *in the steer text* while
nothing charged it, so the push was advisory.

Now, once a negative precedent has actually been **surfaced** for a
`(tool, target)` in a session, using that tool on that target is **refused
before it runs**:

```
PRECEDENT_UNJUSTIFIED — `extract` was reverted or errored on com.foo.Bar in a
case like this, and that precedent was already surfaced to you. Using it anyway
costs a written justification.
  → re-call with precedentOverride="<one line: why this case is different>"
```

The reason is logged and the charge is owed once. Deliberate limits: only a
warning that was *shown* is chargeable (no enforcement by ambush), any internal
error **fails open** so a broken charge never blocks real work, calls with no
surfaced precedent are untouched, and the override is stripped before dispatch
so no tool sees the choke's meta-argument.

## The retired learning kinds are gone

`experience(kind=train | learner_status | observe_edit)` are removed. Sprint 26a
deleted every ML model class, leaving those kinds able to answer nothing but
"retired" — and advertising them told every agent that a capability exists which
does not. An unknown kind now answers with the allowed list, which names the
kinds that do exist. The `/train` command they served is retired in
jawata-studio v3.3.1.
